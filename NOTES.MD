# Notes on Implementation

This project contains two implementations of the IP config parser (`ipconfig_parser`) and a parking calculator solution (`parking_calculator`).

The initial implementations were designed with a focus on modularity, extensibility, and clean separation of concerns, resulting in a more complex but flexible architecture.

After reviewing the actual task requirements, it became clear that several abstractions were not strictly necessary. Instead of refactoring the original code, simplified “light” versions were created to better match the evaluation scope.

---

## ipconfig_parser

Full version:
- main.py
- parser.py
- config.py
- filterFields.py  
→ modular architecture, extended parsing logic, host + adapter handling

Light version:
- main_light.py
- parser_light.py  
→ simplified logic focused only on required adapter fields

---

## parking_calculator

The parking calculator was implemented with maintainability and ease of modification in mind, allowing pricing rules and time-based logic to be easily adjusted without restructuring the core logic.
